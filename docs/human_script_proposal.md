# Script 人类可读格式方案

当前可运行格式是 `configs/competition_script.toml`。它稳定、容易被程序读取，但不像剧本文本。后续建议换成 `.rkscript`，更适合比赛现场人工修改。

## 建议格式

```text
连接 VS 172.29.64.1:7930 超时 3s 重连 1s
连接 3D 192.168.173.2:9303 超时 5s
连接 Bot 192.168.200.1:9552 超时 8s 重试 2

任务 A all:
  VS start;A;all; -> 回复 start;A;all;
  VS A;圆.x;圆.y;方.x;方.y;长.x;长.y;

  对每个物体按顺序:
    MovJ 到 物体.x 物体.y 150 180 0 0
    MovL 到 物体.x 物体.y 123 180 0 0
    Suck 1
    MovJ 到 物体.x 物体.y 150 180 0 0
    MovJ 到 9 10 150 180 0 0
    MovL 到 9 10 当前码垛Z 180 0 0
    Sucl 0

  物体高度:
    圆 = 20
    方 = 25
    长 = 15
  码垛起始Z = 123
  结束 MovJ 到 11 12 150 180 0 0

任务 B:
  VS B;扳手.x;扳手.y;螺母.x;螺母.y;
  发给 3D B;start;
  3D 返回 B;扳手.z;螺母.z;

  GP 扳手.x 扳手.y 100+扳手.z 180 0 0 20 30 180 0 0
  GP 螺母.x 螺母.y 100+螺母.z 180 0 0 50 30 180 0 0 11 12 150 180 0 0
```

## 采用理由

- 像比赛流程说明，不像程序配置。
- 每一行都接近真实 TCP 指令，现场好核对。
- 可以保留变量名，比如 `扳手.x`、`当前码垛Z`。
- 程序可以解析成现在的 TOML 结构，保持运行逻辑稳定。

## 暂定落地顺序

1. 当前版本继续运行 TOML，Web 支持原文编辑和观察。
2. 你确认 `.rkscript` 格式后，增加解析器。
3. Web 改为编辑 `.rkscript`，保存时自动生成内部 TOML 或直接执行。
